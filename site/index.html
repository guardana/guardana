25 rules · two surfaces

What it checks

Static checks are deterministic — no model, no network, no false-positive theatre. Dynamic checks talk to a live endpoint and are graded by a swappable evaluator.

Build-time

17 rules · your laptop, CI, a training box

  • pickle_opcode — code that runs on model load
  • keras_lambda — Lambda-layer RCE in .keras/.h5
  • remote_code_config — auto_map RCE in config.json
  • dependency_risk — torch.load, joblib, unsafe yaml
  • hardcoded_secret — provider keys across 30+ file types
  • hidden_instructions — invisible rules-file backdoors

Runtime

8 rules · against a served model

  • system_prompt_leak — proven by a planted canary
  • injection.ignore_previous — instruction override
  • scenario.gradual_jailbreak — multi-turn escalation
  • scenario.indirect_injection — poisoned retrieval
  • agent.excessive_tool_use — reaches for a destructive tool
  • output.secrets — keys leaking in replies

The differentiator

Did the attack actually work?

Most scanners are good at sending attacks and bad at telling you whether one landed. Guardana treats that judgement as a first-class, swappable, versioned component — so every dynamic finding arrives with an outcome, a rationale, and a number you can gate on.

CRITICAL guardana.prompt.system_prompt_leak.canary

The reply contained the canary token planted in the system prompt — unambiguous evidence of disclosure, not a keyword guess.

Confidence 0.95

Keyword-graded checks misjudge whether an attack succeeded at rates reported as high as 37% — Fujitsu Research, 2024. A scanner that can't tell a refusal from a compliance isn't a security tool.

CI in one step

Gate it on every push

The official Action scans and uploads SARIF, so findings annotate the exact source line in GitHub code scanning. Every finding maps to OWASP LLM Top 10, MITRE ATLAS, and NIST.

// .github/workflows/ai-security.yml
- uses: actions/checkout@v4
- uses: guardana/guardana@v0.1
  with:
    args: --preset ci

Turning a gate on for an existing repo? --write-baseline accepts today's findings with a reason, so only new ones fail the build. There's a pre-commit hook too.
